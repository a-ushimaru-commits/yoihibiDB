# よい日々 売上進捗ダッシュボード 設計書

## 背景・目的

「よい日々」ブランドの1期(2025年6月〜2026年5月)実績を基準に、2期(今期)の売上進捗を月次・日次で確認できる管理ツールを作る。

入力ファイル3種:

1. **1期ベース実績**: `粗利分析_よい日々1期_YYYYMMDD.xlsx` の `詳細明細` シート
   - 列: 月, 販売区分(TV/yahoo/その他/アマゾン/卸/楽天/自社), よい日々(ブランド), 定期/通常, 数量, 売上, 仕入額, 粗利, 粗利率
   - 月次×チャネル×定期通常の粒度で既に集計済み
2. **月次実績**: `商品別収益YYYYMM.xlsx` の `売上明細_提出` シート(受注単位、生データ)
   - よい日々は `ブランド区分` = `"22"` で判定(商品名がMCTオイル/MSMパウダー/スープ・スープ等と一致確認済み)
   - `媒体名` 列が生の販路名(よい日々/楽天よい日々/Amazon/Amazon FBA/BtoB(各社)/TikTok/YAHOOプライムダイレクト等)。7区分へのマッピングが必要
   - 出荷日, 数量, 金額, 仕入金額, 粗利額, 販売区分(定期/通常) 列を使用
3. **日次売上**: `受注_売上一覧表ライト_YYYYMM.csv`(月次実績と同構造、事業部列なし)
   - 同じ `ブランド区分`='22' 判定・媒体名マッピングで抽出。出荷日単位で保持し、当月の日次累積グラフに使う

## アーキテクチャ

- 単一HTMLファイル(`dashboard.html`) + 同梱JSライブラリ(SheetJSをローカル同梱、CDN依存なし)。サーバー不要、ダブルクリックで開いて使う
- ブラウザの `localStorage` に全データを保持。ファイルを閉じても次回開けば取込済みデータが残る
- 「データ書き出し/読み込み」ボタンでJSONバックアップ(PC移行・共有用)

### 取込フロー

- ドラッグ&ドロップ or ファイル選択で、ファイル名・シート構成から種別を自動判別
  - `粗利分析_よい日々1期*.xlsx` → 詳細明細シート → 1期ベース実績として保存(通常は初回のみ、更新されたら再取込で上書き)
  - `商品別収益*.xlsx` → 売上明細_提出シート → ブランド区分='22'抽出 → 媒体名マッピング → 月次実績(月×チャネル×定期通常)に集計して保存
  - `受注_売上一覧表*.csv` → 同様に抽出・マッピング → 日次実績(日付×チャネル×定期通常)として保存
- 同じ対象月(出荷日のYYYY-MM)のデータを再取込した場合、その月のレコードを丸ごと上書き(重複加算防止)
- 媒体名マッピングに存在しない生名称が出てきたら自動で「その他」に割り当て、画面に警告バッジ表示(取込件数・金額付き)。設定パネルでいつでも再マッピング可能

## データモデル

```
Record = {
  yearMonth: "2025-06",
  channel: "TV" | "yahoo" | "その他" | "アマゾン" | "卸" | "楽天" | "自社",
  type: "定期" | "通常",
  sales: number,
  cost: number,
  profit: number,
}
```

- `baseRecords: Record[]` — 1期ベース実績
- `monthlyRecords: Record[]` — 2期月次実績(月次実績xlsxから集計)
- `dailyRecords: (Record & { date: "2025-06-01" })[]` — 2期日次実績
- `targets: { yearMonth, salesTarget, profitTarget }[]` — 月別目標(ユーザーが画面上で入力・編集)
- `mediaMapping: { [rawMediaName]: channel }` — 初期値は以下の推測マッピング。設定パネルで編集可能

初期マッピング(生媒体名パターン → チャネル):

| 生媒体名パターン | チャネル |
|---|---|
| よい日々 | 自社 |
| 楽天よい日々 | 楽天 |
| Amazon, Amazon FBA | アマゾン |
| BtoB(*) | 卸 |
| YAHOO* | yahoo |
| TikTok, Creema, メルカリ, 会報誌, その他不明 | その他 |
| TV系媒体名 | TV |
| 倉庫移動等(売上以外) | 集計対象外(除外) |

## ダッシュボードUI

- **月選択セレクタ**: 表示対象月を切り替え(取込済みの2期月次データがある月のみ選択可)
- **上部KPIカード** x3(売上/粗利/粗利率): 各カードに「1期同月比 ±%」「目標達成率 %」をサブ表示
- **月次推移グラフ**: 2期実績(実線)と1期実績(点線)を売上・粗利それぞれ重ねて表示。目標値があれば横線で表示
- **日次累積グラフ**(選択月のみ): 当月の日次累積売上・粗利と、1期同月を日数按分した目安ペースラインを比較表示(「まだ半月で1期比◯%」を可視化)
- **チャネル別テーブル**: 行=TV/yahoo/その他/アマゾン/卸/楽天/自社、列=売上/粗利/粗利率/1期比/目標比。列ソート可能
- **設定パネル**(折りたたみ式): 媒体名マッピング編集、月別目標入力、データ書き出し/読み込み、取込済みデータのクリア

## スコープ(MVP)

- チャネル別(TV/yahoo/その他/アマゾン/卸/楽天/自社) × 定期/通常 の粒度のみ
- ブランド別(MCTオイル/MSMクリーム等)の内訳は今回スコープ外(将来拡張しやすいデータモデルにはしておく)

## エラーハンドリング

- 取込ファイルの種別判定に失敗した場合、エラーメッセージを表示してユーザーに手動選択させる(自動判別失敗時のフォールバック)
- 想定シート/列が見つからない場合は、該当ファイルの取込を中断し具体的な理由を表示

## テスト方針

- 実データ(本設計書作成時に確認済みの3ファイル)を実際に取込み、Playwrightでブラウザ操作しKPI・グラフ・テーブルの値を目視確認する
- media mapping未知語検出、同月再取込時の上書き動作を確認する
