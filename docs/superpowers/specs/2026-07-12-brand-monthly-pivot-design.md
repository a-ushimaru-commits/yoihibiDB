# 月次推移(ブランド×定期/通常)ピボット表 設計書

## 背景・目的

ユーザーから、参考ファイル「1期2期6月比較.xlsx」の`自社月別`シートのような、月を縦に並べた時系列表示を今のダッシュボードにも追加したいという依頼があった。ヒアリングの結果、以下の要件が確定した:

- 対象は「自社」チャネルに限らず、**全チャネル合計(会社全体)**の月次推移
- ブランド別(MCTオイル、MSMパウダーなど)と定期/通常の**両方**を時系列で見たい
- 表示形式は`自社月別`シートと同じ**広いピボット表**(1行=1か月、列にブランド×定期/通常が並ぶ)

既存のダッシュボードは「対象月」を1つ選んで見る単月ビュー(KPIカード・チャネル別テーブル・ブランド別テーブル)のみを持つ。今回追加するのはこれに並ぶ**新しい時系列セクション**であり、既存機能は変更しない。

## データ源

新しい取込みファイルは不要。既存の`state.baseRecords`(1期)と`state.monthlyRecords`(2期)を**結合**し、1期(2025-06〜2026-05)から2期(2026-06〜)へと続く一本の連続した月次時系列として扱う。両方とも`{ yearMonth, channel, type, brand, sales, cost, profit }`という同じ形状を持つため、単純に配列を連結するだけで扱える。

## 集計: `getBrandMonthlyPivot(state)`

`js/aggregate.js`に新関数を追加する。

```
getBrandMonthlyPivot(state) -> {
  months: string[],   // 例: ['2025-06', '2025-07', ..., '2026-06'] 昇順、baseRecordsとmonthlyRecordsの両方に出現する月の和集合
  brands: string[],   // 全期間で一度でも登場したブランド名、売上合計の降順
  rows: [
    {
      yearMonth: string,
      totalTeikiSales: number, totalTeikiProfit: number,
      totalTsujoSales: number, totalTsujoProfit: number,
      byBrand: {
        [brandName]: { teikiSales: number, teikiProfit: number, tsujoSales: number, tsujoProfit: number }
      }
    },
    ...
  ]
}
```

- 「全体」の集計はチャネルを問わず、その月の該当`type`(定期/通常)のレコードを合計する
- ブランド別集計も同様に、チャネルを問わずその月・ブランド・区分のレコードを合計する
- `brand`が`null`(1期データでブランド列が空欄だった行)のレコードは、全体合計には含めるが`byBrand`の内訳には含めない
- ある月にそのブランドのデータが存在しない場合、該当セルは`{teikiSales: 0, teikiProfit: 0, tsujoSales: 0, tsujoProfit: 0}`とする(欠損ではなく0埋め)

## 表示: `renderBrandMonthlyPivotHTML(pivot)`

`js/ui.js`に新関数を追加する。

- 1行目ヘッダー: 月 / 全体 定期売上 / 全体 定期粗利 / 全体 通常売上 / 全体 通常粗利 / (ブランドごとに) 定期売上 / 定期粗利 / 通常売上 / 通常粗利
- 1行=1か月。ブランド列はブランド名でグループ化した見出し(2段ヘッダー)にする
- ブランドが1つも登場していない場合(データ未取込み等)は「表示できるデータがありません」の空状態メッセージ
- 列数が多くなるため、テーブルを`overflow-x: auto`のコンテナで囲み横スクロール可能にする

## スコープ外(将来拡張)

- ブランド×定期/通常セルごとの1期比較列は追加しない(列数が倍になり実用性が下がるため)。1期→2期の連続した月次行として並べること自体で期をまたいだ推移は確認できる
- 数量(件数)は今のダッシュボードのデータモデルに存在しないため対象外
- 目標値との比較もこの表には含めない(既存のKPIカードで対応済み)

## UI配置

既存の「ブランド別」テーブルの下に、新しい見出し「月次推移(ブランド×定期/通常)」とテーブルを追加する。

## テスト方針

- `getBrandMonthlyPivot`の単体テスト(1期+2期結合、ブランドなしレコードの扱い、欠損月の0埋め)
- `renderBrandMonthlyPivotHTML`の単体テスト(ヘッダー生成、空状態)
- 実データでのPlaywright検証(既存のチャネル別・ブランド別の数値に影響がないことを含む回帰チェック)
